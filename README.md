# Tournament Platform

**Tournament Platform** — це платформа на базі Django для організації хакатонів, ІТ-змагань та турнірів. Це комплексне рішення, що дозволяє керувати подіями: від реєстрації команд до оцінювання робіт журі та автоматичної видачі сертифікатів.

## 🌟 Основний функціонал

* **Багаторівневий доступ**: Ролі користувачів (`Participant`, `Jury`, `Organizer`, `Admin`).
* **Управління турнірами**: Створення турнірів (з налаштуванням дедлайнів, дозволених способів зв'язку, чернеток статусів). 
* **Команди та учасники**: Реєстрація команд, капітанів та учасників з можливістю затвердження організаторами.
* **Завдання та роботи (Submissions)**: Публікація завдань, отримання відповідей у вигляді посилань (GitHub, відео, демо).
* **Оцінювання**: Призначення журі для перевірки конкретних робіт за 4-ма критеріями: Backend, Frontend, UX та Функціональність.
* **Дошка оголошень**: Публікація оголошень для інформування учасників.
* **Сертифікати**: Генерація та видача кастомних сертифікатів (учасникам та переможцям) на базі шаблонів (зображень).

## 🛠 Технологічний стек

* **Стек**: Python 3.12+, Django 5.2.
* **БД**: PostgreSQL (з підтримкою `dj-database-url`).
* **Автентифікація**: Кастомна модель користувача, механізм захисту від брутфорсу (`LoginThrottle`).
* **Обробка статики**: WhiteNoise для сервування статики.
* **Деплой**: Готова інфраструктура для безкоштовного хостингу [Render](https://render.com) (IaC файл `render.yaml` та скрипт збірки `build.sh`).
* **Email сервіси**: Підтримка SMTP (напр., Gmail) або Brevo API.

## 🚀 Встановлення (Локально)

### Вимоги
* Python 3.12+
* PostgreSQL (або SQLite для тестування без урахування `dj-database-url` налаштувань)
* `pip`, `venv`

### Кроки

1. **Клонуйте репозиторій**:
```bash
git clone https://github.com/wodinaz-hub/tournament.git
cd tournament
```

2. **Створіть віртуальне середовище та встановіть залежності**:
```bash
python -m venv venv
source venv/bin/activate  # на Linux/Mac
# venv\Scripts\activate на Windows

pip install -r requirements.txt
```

3. **Налаштуйте змінні середовища**:
Створіть файл `.env` на основі шаблону `.env.example`:
```bash
cp .env.example .env
```
Відредагуйте `.env`, вказавши ваші параметри БД (за замовчуванням: бази `tournament_db`, юзер `postgres`, пароль `postgres`) та `SECRET_KEY`.

4. **Застосуйте міграції**:
```bash
python manage.py migrate
```

5. **Зберіть статику**:
```bash
python manage.py collectstatic --no-input
```

6. **Створіть суперкористувача (Адміністратора)**:
```bash
python manage.py createsuperuser
```

7. **Запустіть сервер локально**:
```bash
python manage.py runserver
```

Додаток буде доступний за адресою `http://127.0.0.1:8000/`.

## 🌐 Деплой на Render.com

Платформа вже повністю підготовлена для розміщення на безкоштовному тарифному плані Render.com (через Blueprint).
1. Прив'яжіть свій GitHub акаунт до Render.
2. В панелі Render створіть новий **Blueprint Instance**.
3. Вкажіть посилання на ваш репозиторій.
4. Render автоматично розгорне базу даних (PostgreSQL) і Web-сервіс (Gunicorn-сервер), використовуючи налаштування з `render.yaml` та скрипт `build.sh`.
